About Static Routes

Static routes allow you to proxy traffic to external services that are not running as Docker containers. Changes are automatically applied to Caddy and saved to the YAML configuration file.

  • Domain: The domain name that will be proxied (e.g., api.example.com)
  • Backend URL: The target service URL (e.g., http://192.168.1.100:8080)
  • Backend Path: Optional path prefix for the backend service
  • Force SSL: Redirect HTTP requests to HTTPS
  • WebSocket: Enable WebSocket support for the route
